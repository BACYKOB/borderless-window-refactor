# HANDOFF — плавная анимация окна (BorderlessWindow)

Краткое описание того, что реализовано для плавного появления/исчезновения и переходов окна.
Всё живёт в `BorderlessWindow.cs` (безрамочное окно, `AllowsTransparency=false`, системная рамка DWM ради чистого ресайза).

## Цель

Убрать резкое появление/исчезновение и вспышки прогрузки подложки Windows; сделать переходы плавным кроссфейдом,
без серого промежуточного этапа, который был раньше.

## Итоговый подход: скриншот-маска («вариант B»)

Главное окно **нельзя** плавно гасить его собственной прозрачностью:
- `AllowsTransparency=true` (per-pixel) роняет производительность — не используем.
- `WS_EX_LAYERED + LWA_ALPHA` (константная альфа) на этом окне **не анимируется** (аппаратный WPF + системная рамка):
  применяются только крайние значения 0/255, промежуточные DWM игнорирует. Проверено — путь тупиковый.

Поэтому анимируется НЕ окно, а **отдельное GDI layered-окно-«маска»** поверх него (GDI-окно альфу тянет нормально):

1. Снимок нужной области экрана (`BitBlt` из `GetDC(NULL)`), кладётся в маску через `UpdateLayeredWindow`.
2. Маска показывается **сразу непрозрачной** (снимок совпадает с тем, что уже на экране — бесшовно; фазы наплыва НЕТ).
3. Под непрозрачной маской выполняется действие (`atPeak`): снятие DWM-cloak / смена `WindowState`.
4. Маска **растворяется** 255→0, открывая новое состояние окна.

Стартовая вспышка подложки гасится **DWM-cloak** (`DWMWA_CLOAK`): окно бинарно убрано из композиции до первого кадра.

## Ключевые методы

- `HideMainForStartup` / `UncloakMain` — cloak/uncloak главного окна (`DWMWA_CLOAK`).
- `StartScreenshotCrossfade(rect, atPeak, gateOnContent, durationMs)` — создаёт маску, снимок, запускает крестфейд.
- `ScreenshotFadeLoop` — поток растворения (альфа по кадрам, `DwmFlush()` после шага; на пике — `atPeak` + gate).
- `UpdateShotAlpha` — обновляет контент+альфу маски через `UpdateLayeredWindow`.
- `CleanupScreenshotMask` — уничтожение окна-маски и освобождение GDI-ресурсов.
- `TryGetMaskRect` — прямоугольник маски = видимые границы окна + запас под тень (см. ниже).
- `EndMask` — прерывание крестфейда и очистка (на закрытии/перезапуске).

## Какие переходы анимируются

- **Появление при старте** — `StartMaskReveal` (cloak → снимок десктопа → uncloak → растворение).
- **Восстановление из трея** — `StartRestoreReveal` (ловится в `OnStateChanged`).
- **Сворачивание** — `AnimateMinimize` (снимок окна → сворачивание под маской → растворение в десктоп).
- **Полноэкранный ⇄ обычный** — `AnimateToWindowState` (маска на весь монитор, смена `WindowState` под ней).
  Только по кнопкам caption и двойному клику; Win+стрелки / snap / перетаскивание идут штатно, без анимации.

## Тень окна

Тень — **системная (DWM)**, не WPF-эффект. Управлять её размером/интенсивностью через WPF нельзя
(попытка своей тени = отключить системную через `NCRENDERING_DISABLED` — ломает тематическую рамку, откатили).

Чтобы тень **анимировалась вместе с окном**, маска накрывает и зону тени. Запас — **несимметричный**
(замеры по скринам: бока тени ~30px, низ ~80px), задаётся константами:
`MaskShadowMarginLeft/Top/Right/Bottom = 48 / 40 / 48 / 140` (физ. px).

Собственную тень окна-маски (Win11 добавляет её WS_POPUP) гасим тремя способами:
`DWMWCP_DONOTROUND` (квадратные углы) + `DWMNCRP_DISABLED` + **оконный регион `SetWindowRgn`** (окно с регионом тень не отбрасывает).

## Синхронизация с контентом (gate)

При раскрытии из cloak растворение начинается **не по таймеру**, а по факту первого представленного кадра с контентом
(`CompositionTarget.Rendering` после `atPeak`, поле `_gateOnRender`, ожидание `_peakGate` с таймаутом `PeakGateTimeoutMs`).
Так подложка скрыта ровно до готовности окна, адаптивно к железу.

## Плавность (easing)

Фаза теперь одна (только растворение), поэтому единая **ease-in-out** кривая `SmoothStep(t) = t*t*(3-2t)`:
мягкий старт и мягкая осадка (скорость 0 на концах). Применяется как `alpha = (1 - SmoothStep(t)) * 255`.
(Прежняя двухфазная схема с симметричным `Ease` по фазам больше не нужна.)

## Длительности (боевые)

- `ScreenshotAppearMs = 120` — появление, восстановление, вход в полноэкранный.
- `ScreenshotDisappearMs = 90` — сворачивание, выход из полноэкранного (исчезновение короче появления).

Пробрасываются в `StartScreenshotCrossfade(..., durationMs)`; для fullscreen выбор по направлению:
`target == Maximized ? ScreenshotAppearMs : ScreenshotDisappearMs`.

## Тюнинг (все константы вверху класса)

- Длительности: `ScreenshotAppearMs`, `ScreenshotDisappearMs`.
- Запас маски под тень: `MaskShadowMarginLeft/Top/Right/Bottom` (если снизу проступит слабый контур — поднять Bottom).
- Общий выключатель анимации: `EnableStartupMask`.
- Кривая плавности: `SmoothStep` (заменить на ease-out `(1 - Ease(t))` при желании «снапнее»).

## Мёртвый код (можно удалить)

Прежняя серая маска больше нигде не вызывается: `StartMaskCrossfade`, `MaskFadeLoop`, а также `Ease` (используется только ими).
Оставлены как неиспользуемые; на компиляцию не влияют.

## Известные ограничения

- Системную DWM-тень нельзя уменьшить/облегчить (своя тень отменена — ломает рамку). Тень оставлена как есть, но анимируется.
- Снимок делается `BitBlt` с экрана: на HDR/защищённом контенте область теоретически может выйти чёрной (на практике не наблюдалось).
